# 需求分析

## 1. 问题分解

### 1.1 知识点粒度定义
- **核心概念**：医学术语、定义、原理（如"动作电位"、"炎症"）
- **生理机制**：生命活动的过程和规律（如"细胞信号转导"）
- **病理变化**：疾病状态下的异常改变（如"细胞坏死"）
- **临床表现**：疾病的症状和体征
- **治疗方法**：药物、手术等干预手段
- **解剖结构**：器官、组织、细胞的形态结构

### 1.2 重复判定标准
- **语义相似度阈值**：0.75（基于BGE-small-zh embedding的余弦相似度）
- **LLM判断**：对高相似度候选对进行人工判断，确认是否等价
- **判定维度**：概念名称、定义内容、应用场景

### 1.3 教学连贯性保障
- 保留前置依赖关系：如果知识点A是B的前置依赖，两者都保留
- 保留知识链路：确保学习路径不断裂
- 教师可干预：通过对话界面调整整合决策

### 1.4 压缩比计算方式
- 原始总字数 = 所有教材章节内容字数之和
- 整合后字数 = 保留知识点的定义和描述字数之和
- 压缩比 = 整合后字数 / 原始总字数
- 目标：压缩比 ≤ 30%

### 1.5 RAG分块策略选择依据
- **分块大小**：600字（平衡上下文完整性和检索精度）
- **重叠窗口**：80字（防止知识点在边界处被截断）
- **选择理由**：
  - 医学教材段落通常200-800字，600字能覆盖一个完整知识点
  - 过小的分块（<300字）会丢失上下文，导致检索不准确
  - 过大的分块（>1000字）会包含过多无关信息，降低检索精度
  - 80字重叠确保边界知识点在两个chunk中都有完整表述

## 2. 子问题分析

### 2.1 文件解析
- PDF解析使用PyMuPDF，支持逐页处理大文件
- 章节识别：正则匹配"第X章"模式 + 字体大小判断
- 页眉页脚过滤：基于位置和长度的启发式规则

### 2.2 知识提取
- 使用DeepSeek API进行LLM调用
- 每个章节独立提取，避免上下文过长
- 结构化JSON输出，便于后续处理

### 2.3 知识图谱构建
- 节点：知识点（概念、定义、分类、来源）
- 边：关系（前置依赖、并列、包含、应用）
- 使用D3.js力导向图进行可视化

### 2.4 跨教材整合
- 第一步：Embedding计算语义相似度
- 第二步：LLM判断候选对是否等价
- 第三步：执行整合决策（合并/保留/删除）
- 第四步：生成整合报告

### 2.5 RAG问答
- 使用FAISS进行向量检索
- BGE-small-zh作为Embedding模型
- 检索top-5最相关的chunk
- LLM生成带引用的回答
